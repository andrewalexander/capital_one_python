nessie.account module¶

class nessie.account.Account¶

Account class - get/update account information.

Some of these functions require a customer ID that can be retrieved from the :ref: nessie.customer.Customer class.

create_account(cust_id, request_data)¶

Creates a new account for cust_id using the details in request_data

Format for POST request:

{
    'type': 'Checking' | 'Credit Card',
    'nickname': 'string',
    'rewards': int,
    'balance': int
}

Format of response Object:

{
    "code": 201,
    "message": "Account created",
    "objectCreated": {
        "type": "Credit Card",
        "nickname": "Brand New test Account",
        "rewards": 0,
        "balance": 0,
        "customer_id": "string",
        "_id": "string"
    }
 }
Parameters:
  • cust_id – The owner of the new account we are creating
  • request_data – The dict containing the details of the new account following the format above
Returns:

dict with status code (201) and response object from Nessie backend with details about created object

Delete the account specified by acc_id

Parameters:acc_id – The account ID to delete
Returns:dict with status code (204) and empty response

update_deposit(deposit_id, deposit)¶

Update metadata associated with a deposit

This will return a 404 if the deposit status is ‘executed’ or ‘cancelled’, so it should only be called immediately after a ‘create’.

Format of PUT request:

    'medium': 'balance' | 'rewards',
    'amount': int,
    'description': 'string'
}
